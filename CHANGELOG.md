
##  НОВЫЕ ФАЙЛЫ

### Инфраструктура:
1. **logger.py** 
   - Система логирования со временем
   - Логирование в консоль и файлы
   - Отдельные логгеры для каждого модуля
   - Функции для логирования специальных событий (открытия двери, ошибок)

2. **storage.py** 
   - Управление хранилищем скриншотов
   - Генерация путей формата год/месяц/день
   - Сохранение информации о жестах в имени файла
   - Функции для получения данных за день/месяц
   - Парсинг информации из имён файлов

3. **config.py** 
   - Централизованная конфигурация
   - Загрузка из .env файла
   - Константы для всех модулей

4. **auth.py** 
   - Верификация пароля
   - Управление сеансами
   - Проверка валидности сеанса
   - Декоратор для защиты эндпоинтов

5. **database.py**
   - Класс Database для работы с PostgreSQL
   - Методы: execute, fetch_one, fetch_all
   - Функции для вставки и получения данных
   - Обработка ошибок и логирование

### Веб-приложение:
6. **app.py** 
   - Flask приложение
   - Маршруты для авторизации и просмотра
   - API endpoints для фронтенда
   - Обработчики ошибок

7. **run.py**
   - Главный entry point приложения
   - Запуск всех компонентов в правильном порядке

### Веб-интерфейс:
8. **templates/login.html**
   - Красивая страница входа с градиентом
   - Форма для ввода логина и пароля
   - Отображение ошибок входа
   - Тестовые учётные данные

9. **templates/live.html**
   - Просмотр текущего видеопотока
   - Статистика обработки (FPS, кадры)
   - Навигация по страницам

10. **templates/calendar.html** 
    - Интерактивный календарь месяца
    - Выделение дней с открытиями
    - Навигация по месяцам/годам
    - Клик на день переходит на детальный просмотр

11. **templates/day.html** 
    - Просмотр всех открытий за день
    - Большие скриншоты (300+ пикселей)
    - Время каждого открытия
    - Список использованных жестов
    - Код ответа API

12. **templates/error.html** 
    - Страница для отображения ошибок
    - Кнопка возврата на главную

13. **static/css/style.css** 
    - Базовые стили для всех страниц
    - Адаптивный дизайн (mobile-friendly)
    - Gradients и переходы
    - Responsive grid layouts

14. **static/js/live.js** 
    - Обновление лайв-стрима каждые 500ms
    - Обновление статистики каждые 1000ms
    - Обработка ошибок

## ПРИМЕЧАНИЯ

- Все временные метки в логах используют формат `YYYY-MM-DD HH:MM:SS`
- Скриншоты автоматически организуются в структуру год/месяц/день
- Имена файлов содержат информацию о жестах для быстрого анализа
- Система предупреждает об ошибках с подробной информацией
- Для production используйте настоящий SECRET_KEY и HTTPS
